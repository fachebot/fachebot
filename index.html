使用blockc0de发送Telegram消息

发表于 2022-01-16 分类于 blockc0de

前言

通过前面《使用blockc0de实时监控链上合约事件》和《使用blockc0de解码合约事件日志》两篇文章的学习我们知道了如何监听和解析合约事件。今天我们来学习不一样的东西，如何通过Telegram Bot给用户发送通知。这是一个非常有用的功能，试想一下如果你编写了一个Graph监控到了巨额的USDT转账，然后自动通过Telegram Bot发送消息通知你，整个流程能够完全在Graph上面实现，会非常便捷。

创建Bot

首先我们需要创建一个Telegram Bot，在 Telegram APP里面搜索用户 @BotFather，通过和这个Bot进行聊天我们就可以创建自己的机器人了。然后向BotFather发送一条 /newbot 命令创建一个新机器人。BotFather 会要求你输入机器人的名称和用户名，最后会为你的新机器人生成身份验证令牌(access token)。

我们需要将BotFather返回的 access token(截图红框部分) 保存下来，稍微会用到。

获取chatId

如果想要使用Telegram API向某人或某个群组发送消息就需要知道对方的 chatId。获取 chatId 的方式也很简单，在 Telegram APP里面搜索用户 @getidsbot，向它发送命令 /start 它就会将你的 chatId 告诉你，下图中红框部分就是我的 chatId。

如果想要获取群组的 chatId，则需要将 @getidsbot 邀请进群，它会自动发送一条包含群组 chatId 的消息，如下图所示：

编写Graph

准备工作已经就绪了，现在我们可以开始编写Graph了，打开blockde在线编辑器：

  1. 点击左侧节点列表中 Telegram Bot 节点， Telegram Bot 节点用于创建一个Bot实例。它接受一个名为 AccessToken 的参数，这个参数的值就是我们在Telegram中创建Bot时由BotFather返回给我们的。我们需要创建一个 String 节点保存 AccessToken，然后将 String 节点与Telegram Bot 节点的 AccessToken 的参数连接。

  2. 点击左侧节点列表中 Send Telegram Bot 节点，Send Telegram Bot 节点用于控制Telegram Bot发送一条消息。Send Telegram Bot 节点接受三个参数：TelegramBot、ChatId 和 Message。首先我们将 Telegram Bot 节点的 Out 与 Send Telegram Bot 节点的 In 连接。参数 TelegramBot 表示 Bot 实例，所以直接将上面创建的 Telegram Bot 节点与它连接。ChatId 表示接受方，这里直接直接创建一个 String 节点，填入上面通过 @getidsbot 获取到的 chatId 填入，然后与 Send Telegram Bot 节点的 ChatId 参数连接。最后创建一个 String 节点，输入想要发送的消息，然后与 Send Telegram Bot 节点的 Message 参数连接。

运行Graph

点击顶部菜单栏 Execution/Launch 按钮上传Graph至服务器并执行，如果一切顺利的话你应该就能在Telegram上面收到来自Graph发送的消息了。

使用blockc0de解码合约事件日志

发表于 2022-01-12 更新于 2022-01-16 分类于 blockc0de

前言

通过上一篇文章使用blockc0de实时监控链上合约事件我们学习了如何编写一个Graph用于监控链上合约事件，在文章末尾留下了一个问题，如何将事件日志转换为可读的信息？现在我们就来讲解如何使用事件日志解码器来解码原始的事件信息，这样我们就能清楚的知道事件日志包含的信息，从何出判断链上智能合约产生了什么操作。

合约ABI

ABI是合约接口的说明，内容包括合约的接口列表、接口名称、参数名称、参数类型、返回类型等，这些信息以JSON格式保存。通常项目方也会把合约源代码公布到区块浏览器，其中就包含了ABI信息，我们可以调用etherscan的API接口查询合约的ABI(如果项目方公布的话)。在浏览器中打开地址 https://api.etherscan.io/api?module=contract&action=getabi&address=0xdac17f958d2ee523a2206206994597c13d831ec7 会返回USDT合约的ABI，0xdac17f958d2ee523a2206206994597c13d831ec7 是USDT合约的地址，可以将它替换为其它合约的地址查询它的ABI。

所以我们在解码合约事件日志之前必须要知道合约ABI才行，ERC20代币有标准的合约ABI，blockc0de已经内置了，我们将会学习如何使用它。另外blockc0de还支持自定义ABI，用于满足用户解析不同合约的需求。

编写Graph

现在我们来继续编写上次的Graph，将 On Ethereum Event Log 节点的输出参数 EventLog 解析并显示为可读的信息。

  1. 点击左侧节点列表中 Ethereum Log Decoder 节点，Ethereum Log Decoder 节点接受两个参数：1. EventLog 参数表示事件日志信息，我们将 On Ethereum Event Log 的输出参数 EventLog 与它进行连接。Abi 参数表示合约接口的说明，我们可以自己创建一个全新的ABI节点并输入JSON格式的内容，也可以使用内置的ABI节点。最后将 On Ethereum Event Log 节点的 Out 与 Ethereum Log Decoder 节点的 In 连接，每当接收到新的事件日志时，都会去执行 Ethereum Log Decoder 节点。

  2. blockc0de内置了ERC20代币的ABI信息，点击左侧节点列表中 ERC20 ABI 节点，将输出参数 Abi 与 Ethereum Log Decoder 节点的输入参数 Abi 连接。

  3. 创建一个 Print 节点用于打印解析后的数据，将 Ethereum Log Decoder 节点的 Out 与 Print 节点的 In 连接，EventLog 与 Message 连接。

运行Graph

点击顶部菜单栏 Execution/Launch 按钮上传Graph至服务器并执行，如果监听到合约事件就会在 Terminal 处输出解析后事件信息。

我们可以看到输出内容为JSON格式，字段 event 表示事件名称，字段 inputs 表示事件的参数列表。ERC-20 TOKEN STANDARD 定义了两个事件：

Transfer 是转账事件，Approval 是授权事件。Transfer 有三个参数，_from 表示转账者地址，_to 表示接收者地址，_value 表示转账金额，这三个参数与输出 JSON 中的 inputs 对应。

最后问题

最后有没有感觉转账金额非常大？这是因为计量单位的问题，我们还需要对转账金额进行一次单位转换，blockc0de 提供了 Wei To Decimals 节点处理单位转换，这个内容我们留到以后再讲解吧。

使用blockc0de实时监控链上合约事件

发表于 2022-01-10 更新于 2022-01-16 分类于 blockc0de

前言

在上一篇文章中我们学习了如何编写第一个Graph，并在终端打印输出 Hello World。现在我们开始学习如何实时监听链上合约事件，是不是感觉到进展太快？因为之前我们还在学习打印输出Hello World，现在就开始区块链编程之旅了。blockc0de 0代码区块链编程工具就是这么神奇，让没有编程基础的人也能够快速玩转区块链，现在我们开始吧。

合约事件

首先我先介绍一下什么是合约事件。我尽量说得直白一点，智能合约可以在发生某个动作后进行事件触发，然后将一些参数保存到交易日志中，这些日志与合约地址相关联。比如我们在以太坊网络转账USDT代币，转账成功后USDT的智能合约就会写入一条 Transfer(address,address,uint256) 事件日志(ERC20代币都是这样)。那事件日志有什么用呢？常用的用途就是我们可以根据事件日志判断链上发生了什么事情，比如上面的 Transfer(address,address,uint256) 事件，根据它我们就可以知道谁给谁转了多少个USDT。

编写Graph

下面我们开始编写一个Graph用于监控以太坊主网上USDT合约的事件日志。

  1. 首先点击左侧节点列表中 Ethereum Connector 节点，Ethereum Connector 节点用于连接到以太坊 RPC 节点，当然也可以连接到其它 EVM 兼容链，但是你需要指定 Url 和 SocketUrl 参数。如果我们不设置这两个参数就会默认连接到以太坊主网。

  2. 点击左侧节点列表中 On Ethereum Event Log，On Ethereum Event Log 节点会监听合约事件，当事件触发的时候会去执行 Out 连接的节点。它接受两个参数：Connection 表示连接器，我们将之前创建的 Ethereum Connector 节点与它进行连接。Contract 表示要监听的合约地址。

  3. 点击左侧节点列表中 String 节点，然后在 String 节点中输入USDT的合约地址：0xdac17f958d2ee523a2206206994597c13d831ec7。再将输出参数 Value 连接到 On Ethereum Event Log 节点的输入参数 Contract。

  4. 为了能够在监控到合约事件后看到事件的内容，我们还是需要将事件信息打印出来。还记得 Hello World 的例子吗？我们创建一个 Print 节点，并将 On Ethereum Event Log 节点的 Out 与 Print 节点的 In 连接， On Ethereum Event Log 节点的输出参数 EventLog 与 Print 节点的输入参数 Message 连接。

短短几个步骤，我们的Graph已经编写完成了，接下来运行起来看看效果吧。

运行Graph

点击顶部菜单栏 Execution/Launch 按钮上传Graph至服务器并执行，如果监听到合约事件就会在 Terminal 处输出事件信息。

可以看到虽然输出很多事件日志信息，但是完全看不明白什么意思😂。不用着急，下次我们继续学习如何解析这些数据，当然解析这些数据一如既往地简单，因为 blockc0de 存在的意义就是为了让没有编程基础的人也能够快速玩转区块链。

Hello blockc0de

发表于 2022-01-06 更新于 2022-01-16 分类于 blockc0de

项目介绍

blockc0de 是一款支持 0 代码开发区块链应用的工具，用户通过连接图形节点来处理业务逻辑的所有需求，而不需要任何编码技能。一旦用户准备好部署Graph，可以将其直接上传至网络并运行，借助 blockc0de 用户可以建立可视化、自动化任务。

任何人都可以在 blockc0de 中创建Graph，Graph由多个不同类型的节点和连接组成。每个节点至少有一个接受要处理的值的输入和一个产生结果的输出。将一个输出连接到一个输入建立一个数据路径，使一个节点能够将数据输入到另一个节点。用这个简单而有力的原则可以实现任何复杂的业务逻辑。例如：监听链上鲸鱼转账并通过Telegram或者Twitter发送通知。

Hello World

Hello World是一个最著名的程序。对每一位程序员来说，这个程序几乎是每一门编程语言中的第一个示例程序。下面我也试着用 blockc0de 实现一个Hello World的Graph。

  1. 首先打开 blockc0de 在线 IDE，点击右上角 Connect to wallet 使用 MetaMask 钱包进行登录。

  2. 点击左侧节点列表里面的 Entry Pont 节点，会自动在Graph上面创建一个 Entry Pont 节点，Entry Pont 是Graph执行的入口，类型其它编程语言中的 main() 函数。

  3. 创建一个 Print 节点，并将 Entry Pont 节点的 Out 与 Print 节点的 In 进行连接，表示 Entry Pont 执行完成之后应立即执行 Print 节点。Print 节点可以将 Message 参数打印输出。

  4. 我们可以看到 Print 节点还需要接收一个名为 Message 的参数，现在我们再创建一个 String 节点，它可以存储一个字符串类型的值。创建完成后我们输入文本 Hello World，并将它与 Print 节点的 Message 参数进行连接。

  5. 现在我们的第一个Graph已经编写完成了，点击顶部菜单栏 Execution/Launch 按钮即可上传至服务器并执行。

  6. 最后在右下角 Terminal 处输出 Hello World，大功告成。

当然 blockc0de 还能在不编写任何代码的情况下实现更加复杂的业务逻辑，我们留着以后慢慢探索吧。
